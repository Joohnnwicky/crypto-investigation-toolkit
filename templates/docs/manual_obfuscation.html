obfuscation_detector_manual.sh

操作步骤、结果解释、API说明与案例演示

>> 1. 操作步骤

  • 1. 配置Etherscan API密钥（首次使用必须配置）。
  • 2. 输入待分析的以太坊地址或交易哈希。
  • 3. 选择检测类型：三明治攻击、闪电贷攻击、粉尘攻击、协议漏洞利用。
  • 4. 点击"开始检测"，系统将分析地址的交易行为模式。
  • 5. 滚动查看检测结果，包括攻击类型、置信度、涉及交易。
  • 6. 导出检测报告（JSON/CSV/PDF），包含详细证据链。

>> 2. 结果解释

攻击类型说明

  • >> 三明治攻击：在用户交易前后插入交易获利，影响交易价格
  • >> 闪电贷攻击：利用无抵押借贷在同一交易内完成攻击
  • >> 粉尘攻击：发送微小金额到目标地址以追踪其活动
  • >> 协议漏洞利用：利用智能合约漏洞进行攻击

置信度说明

  • >> 高置信度（HIGH）：检测到明确的攻击特征模式
  • >> 中置信度（MEDIUM）：存在可疑特征，需要进一步验证
  • >> 低置信度（LOW）：特征不明显，可能为误判

检测结果

  • >> 攻击类型：识别出的攻击种类
  • >> 涉及交易：参与攻击的交易的哈希列表
  • >> 攻击金额：估算的攻击获利或影响金额
  • >> 时间线：攻击发生的时间顺序

>> 3. API密钥说明

需要Etherscan API密钥，ETH链检测专用。

本工具仅支持以太坊链上的混淆攻击检测，需要Etherscan API获取交易数据。

Etherscan API获取

  • 1. 访问 Etherscan官网 并注册账户
  • 2. 登录后进入API-KEYs管理页面
  • 3. 点击"Add"创建新的API密钥
  • 4. 复制生成的API Key并配置到本工具

注意：本工具目前仅支持以太坊链（ETH）的攻击检测，暂不支持其他链。

>> 4. 案例演示

案例背景

某DEX用户报告交易滑点异常，怀疑遭遇三明治攻击，需要分析交易行为。

操作过程

  • 1. 输入可疑攻击者地址：0x1234...5678
  • 2. 选择"三明治攻击"检测类型
  • 3. 系统分析该地址的MEV相关交易
  • 4. 检测结果：高置信度检测到15次三明治攻击
  • 5. 导出详细报告，包含攻击交易列表和获利金额

结论

成功识别出MEV攻击者的攻击行为，为用户投诉提供了技术证据支持。
