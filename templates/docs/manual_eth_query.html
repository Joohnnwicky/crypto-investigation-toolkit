ETH交易查询 - 使用手册

操作步骤、结果解释、API说明与案例演示

1. 操作步骤

  1. 在设置页面配置Etherscan API密钥（首次使用必须配置）。
  2. 输入待查询的以太坊钱包地址（以0x开头的42位地址）。
  3. 选择查询类型：普通交易、内部交易或ERC-20代币交易。
  4. 点击"开始查询"按钮，系统将通过Etherscan API获取交易数据。
  5. 查看交易列表、交易详情，支持导出JSON或CSV格式报告。

2. 结果解释

交易列表信息

  • 交易哈希：唯一标识该笔交易的64位十六进制字符串
  • 区块号：交易被打包确认的区块高度
  • 时间戳：交易确认的具体时间
  • 发送方/接收方：交易的from和to地址
  • 金额：转账的ETH数量（单位：Ether）

跨链桥识别

  • 跨链桥标识：当交易涉及已知跨链桥合约时，系统自动标注
  • 桥名称：如Multichain、Stargate、Across等
  • 目标链：显示资金跨链的目标区块链

ERC-20代币交易

  • 代币符号：如USDT、USDC、LINK等
  • 代币合约：该代币的智能合约地址
  • 转账金额：根据代币精度显示的实际数量

3. API密钥说明

需要Etherscan API密钥，请在下方API获取指南查看注册方法。

获取步骤：

  1. 访问 Etherscan官网 并注册账户
  2. 登录后进入API-KEYs管理页面
  3. 点击"Add"创建新的API密钥
  4. 复制生成的API Key并配置到本工具的设置页面

免费版API每秒最多5次请求，每日10万次调用限制。如需更高频率，可购买付费套餐。

4. 案例演示

案例背景

调查人员需要追踪一笔可疑的以太坊交易，了解资金来源和去向。

操作过程

  1. 输入可疑地址：0x742d35Cc6634C0532925a3b844Bc454e4438f44e
  2. 选择查询类型：普通交易
  3. 查看交易历史：发现该地址与多个DEX合约交互
  4. 切换ERC-20代币查询：发现大额USDT转账
  5. 导出CSV报告进行进一步分析

结论

通过交易查询，追踪到资金最终流入某跨链桥合约，可结合其他工具进一步追踪跨链资金。
