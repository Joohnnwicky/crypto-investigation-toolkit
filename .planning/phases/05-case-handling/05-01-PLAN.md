<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> Backend | Address strings and ETH API key cross this boundary |
| Backend -> External APIs | Tronscan/Etherscan/Blockstream API calls with user-provided addresses |
| Frontend -> sessionStorage | Monitor results stored temporarily for tool interconnection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Address input | mitigate | Validate address format using chain_detector regex patterns before API calls |
| T-05-02 | Information Disclosure | ETH API key | mitigate | Per-query input only (D-06), never stored in session or database, transmitted via POST body only |
| T-05-03 | Denial of Service | External API calls | accept | Manual refresh only (D-03), no auto-polling, user-controlled rate limiting implicit |
| T-05-04 | Tampering | sessionStorage export | accept | Temporary storage only, cleared on browser close, no sensitive data exposure beyond address lists |
</threat_model>
