## 多链监控定义

### 监控触发机制

| Option | Description | Selected |
|--------|-------------|----------|
| 手动刷新 | 用户点击刷新按钮时查询各链API，不自动轮询。最简单，符合"零配置"理念。 | ✓ |
| 定时轮询 | 页面打开时每隔固定间隔（如30秒）自动查询，需考虑API限流和资源消耗。 | |
| 手动对比变化 | 仅记录监控开始时的地址状态，用户手动点击"检查变化"对比当前状态与初始状态。 | |

**User's choice:** 手动刷新（推荐）
**Notes:** 本地工具无后台服务，手动刷新是最简单可靠的方式

### 地址输入格式

| Option | Description | Selected |
|--------|-------------|----------|
| 多地址输入 | 文本域输入，每行一个地址，系统自动检测链类型（TRON/ETH/BTC）。与地址聚类工具一致。 | ✓ |
| 单地址输入 | 单地址输入，用户选择链类型。更适合单独监控场景。 | |

**User's choice:** 多地址输入（推荐）
**Notes:** 与地址聚类一致，最多10个地址

### 监控状态展示

| Option | Description | Selected |
|--------|-------------|----------|
| 状态卡片 | 显示各地址的当前余额、最近交易数、最后活跃时间。每次刷新更新。 | ✓ |
| 变化提示 | 仅显示自上次刷新后是否有新交易（有/无变化）。更简洁。 | |
| 交易列表 | 显示完整交易列表，包括交易哈希、金额、时间。信息更详细但占用空间。 | |

**User's choice:** 状态卡片（推荐）
**Notes:** 每个地址一个卡片，显示余额、交易数、最后活跃时间

### 监控数量上限

| Option | Description | Selected |
|--------|-------------|----------|
| 最多10个 | 与地址聚类一致，防止API请求过多导致限流问题。 | ✓ |
| 最多20个 | 允许更多地址，但需注意各链API限流限制。 | |
| 不限制 | 不限制，用户自行负责API限流问题。 | |

**User's choice:** 最多10个（推荐）

### ETH API密钥输入

| Option | Description | Selected |
|--------|-------------|----------|
| 统一输入框 | 监控页面顶部一个ETH密钥输入框，所有ETH地址共用。与ETH交易查询一致。 | ✓ |
| 每地址输入 | 每个ETH地址单独输入密钥，更灵活但更复杂。 | |

**User's choice:** 统一输入框（推荐）

## 混淆攻击范围

### 攻击类型识别

| Option | Description | Selected |
|--------|-------------|----------|
| Sandwich攻击 | DEX三明治攻击：大户在用户交易前后插入交易，操纵价格套利。 | ✓ |
| 闪电贷攻击 | 利用闪电贷借出大量资金，操纵市场后瞬间还款。常见于DeFi攻击。 | ✓ |
| 粉尘攻击 | 向大量地址发送微量代币，用于追踪地址关联。诈骗常用手法。 | ✓ |
| 协议漏洞攻击 | 利用协议设计缺陷盗取资金，如重入攻击、价格操纵等。 | ✓ |

**User's choice:** 全选（4种攻击类型）
**Notes:** Sandwich、闪电贷、粉尘、协议漏洞全部识别

### 检测逻辑实现

| Option | Description | Selected |
|--------|-------------|----------|
| 规则匹配 | 根据已知攻击特征规则匹配（如闪电贷特征：单笔交易内借贷和还款）。简单可靠。 | ✓ |
| 机器学习模型 | 使用机器学习模型识别异常模式。复杂，本地工具不适用。 | |

**User's choice:** 规则匹配（推荐）
**Notes:** 基于已知攻击特征编码规则，简单可靠

### 检测结果展示

| Option | Description | Selected |
|--------|-------------|----------|
| 攻击卡片 | 每种攻击类型一个卡片，显示攻击详情、置信度、影响地址。与混币追踪一致。 | ✓ |
| 状态指示 | 仅显示是否检测到攻击（有/无），点击查看详情。更简洁。 | |
| 攻击链分析图 | 显示完整的攻击链分析图，包括攻击流程、资金流向。信息详细但复杂。 | |

**User's choice:** 攻击卡片（推荐）
**Notes:** 置信度用颜色区分（HIGH/MEDIUM/LOW）

### 输入方式

| Option | Description | Selected |
|--------|-------------|----------|
| 地址输入 | 输入地址，系统查询该地址的最近交易并检测攻击痕迹。 | ✓ |
| 交易哈希输入 | 输入交易哈希，系统解析单笔交易是否为攻击交易。更精确但范围更窄。 | |
| 两者都支持 | 同时支持地址和交易哈希两种输入方式，用户选择。灵活但界面复杂。 | |

**User's choice:** 地址输入（推荐）

### 检测链范围

| Option | Description | Selected |
|--------|-------------|----------|
| 仅ETH | 仅检测ETH链上的攻击（Uniswap等DEX攻击）。符合现有ETH工具支持范围。 | ✓ |
| ETH + TRON | 同时检测ETH和TRON链上的攻击。TRON攻击特征较少。 | |

**User's choice:** 仅ETH（推荐）

## 资产追回模板

### 模板定位

| Option | Description | Selected |
|--------|-------------|----------|
| 独立模板 | 跨境协查是国际协作请求模板，资产追回是冻结申请模板。定位不同，内容不同。 | ✓ |
| 合并为跨境协查子功能 | 资产追回是跨境协查的子功能，冻结申请是协查请求的一部分。合并开发。 | |

**User's choice:** 独立模板（推荐）
**Notes:** 与跨境协查不同定位，冻结申请专用模板

### 模板字段

| Option | Description | Selected |
|--------|-------------|----------|
| 案件基本信息 | 案件编号、调查机构、联系人信息 | ✓ |
| 冻结对象信息 | 涉案地址列表、链类型、冻结金额 | ✓ |
| 冻结理由说明 | 可疑行为描述、资金来源说明、冻结必要性 | ✓ |
| 冻结条款 | 冻结期限、解除冻结条件、联系人 | ✓ |

**User's choice:** 全选（4类字段）

### 输出格式

| Option | Description | Selected |
|--------|-------------|----------|
| 文本复制 | HTML页面显示，用户复制文本导出。与跨境协查一致。简单实用。 | ✓ |
| PDF导出 | 导出为PDF文件，格式化打印。需额外PDF库，Phase 6会实现PDF导出。 | |
| 两者都支持（分阶段） | 同时支持文本复制和PDF导出。Phase 5仅文本复制，Phase 6追加PDF。 | |

**User's choice:** 文本复制（推荐）
**Notes:** PDF导出在Phase 6实现

### 输入流程

| Option | Description | Selected |
|--------|-------------|----------|
| 分步表单 | 案件信息 → 地址选择 → 模板生成。与跨境协查一致，用户熟悉。 | ✓ |
| 单页表单 | 所有信息在一个页面填写，一次性生成。更紧凑但信息量大。 | |

**User's choice:** 分步表单（推荐）
**Notes:** 3步流程，与跨境协查一致
