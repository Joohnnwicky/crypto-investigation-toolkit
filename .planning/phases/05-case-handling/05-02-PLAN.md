   - For each block with >= 3 DEX transactions (to DEX_ROUTERS addresses):

     - Return {'type': 'Sandwich', 'confidence': 'MEDIUM', 'block': block, 'tx_count': N, 'details': '区块 X 内有 N 笔DEX交易，可能存在三明治攻击'}
   - Use HIGH confidence if same token pair and significant price delta detected (requires log analysis)

3. Implement detect_flash_loan_attack(txs, api_key) per D-15, D-19:
   - For each transaction, check if value > 100 ETH or involves known flash loan patterns
   - Check transaction logs for 'Borrow' and 'Repay' events (requires get_transaction_logs - optional enhancement)
   - Return {'type': 'Flash Loan', 'confidence': 'HIGH', 'tx_hash': hash, 'value': amount, 'details': '交易内含借贷和还款'}

4. Implement detect_dusting_attack(txs) per D-16, D-21:
   - Filter outgoing transactions with value < DUST_THRESHOLD (converted to ETH)
   - Count unique recipient addresses
   - If >= 10 unique recipients with tiny amounts: return {'type': 'Dusting', 'confidence': 'HIGH', 'tx_count': N, 'recipients': M, 'details': '发现 N 笔小额转账，涉及 M 个地址'}
   - Use HIGH confidence if >= 50 recipients

5. Implement detect_protocol_vulnerability(txs) per D-17, D-22:
   - Check for failed transactions (isError == '1') with high value > 10 ETH
   - Return {'type': 'Protocol Vulnerability', 'confidence': 'LOW', 'tx_hash': hash, 'details': '高价值交易失败，可能涉及协议漏洞尝试'}

   - Sort results by confidence (HIGH first, following mixer_tracker pattern)
   - Return {'success': True, 'address': address, 'attack_cards': [...], 'message': '未发现攻击痕迹'}

Use logger.warning for detection errors, logger.info for successful detections.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-06 | Information Disclosure | ETH API key | mitigate | Per-query input only (D-25), transmitted via POST body, never persisted |
| T-05-07 | Spoofing | Detection results | accept | Rule-based heuristics may produce false positives; user judgment required |
