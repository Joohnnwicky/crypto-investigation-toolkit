Create modules/tron/behavior_analyzer.py implementing 4 behavior patterns per D-01:

1. `analyze_first_funding_source(address: str, transfers: List[Dict]) -> Dict`
   - Sort transfers by timestamp ascending (oldest first)
   - Find first incoming transfer where to_address == address
   - Return: {funder_address, first_transfer_time, first_amount, funder_tx_count}

2. `analyze_transfer_patterns(address: str, transfers: List[Dict]) -> Dict`
   - Calculate total_in (sum of incoming amounts), total_out (sum of outgoing)
   - Calculate in_out_ratio = total_in / total_out (handle division by zero)
   - Calculate avg_transfer_interval_hours from timestamps
   - Return: {total_in, total_out, in_out_ratio, avg_transfer_interval_hours, transfer_count_in, transfer_count_out}

3. `analyze_address_relationships(address: str, transfers: List[Dict]) -> Dict`
   - Count interactions per counterparty address
   - Calculate total amount per counterparty
   - Return top 5 counterparties sorted by interaction_count desc
   - Return: {top_counterparties: [{address, interaction_count, total_amount}], unique_addresses_interacted}

4. `analyze_activity_timeline(address: str, transfers: List[Dict]) -> Dict`
   - Find first_activity (min timestamp), last_activity (max timestamp)
   - Calculate active_days = (last - first) / (1000 * 3600 * 24)
   - Identify peak_activity_period (most transfers in 7-day window)
   - Return: {first_activity, last_activity, active_days, peak_activity_period}

5. `analyze_behavior_web(address: str) -> Dict[str, Any]` (wrapper per D-02)
   - Validate address with is_valid_tron_address (copy pattern from suspicious_analyzer.py)
   - Call get_trc20_transfers(address, limit=100) (100 transfers per A3 assumption)
   - Call all 4 analysis functions
   - Return: {success, address, behaviors: {funding_source, transfer_patterns, relationships, timeline}, error}
   - Handle empty transfers: return status='无记录' for each behavior
</action>
